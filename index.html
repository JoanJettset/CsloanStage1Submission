HTML and the Web

HTML is an Important Part of How the Internet Works

HTML is Hyper Text Markup Language. It is the basis of most of the documents on the web. HTML documents contain text content, which is text you see when you are looking at the web page, and markup, which is the instructions for how the text should be displayed on the webpage.

When you use your computer to go to a website, your computer uses something called Hyptertext Transfer Protocol or HTTP to send requests to servers. These servers respond by finding the appropriate HTML documents, and you see the web page you requested when you clicked a link or entered a URL. Your browser reads and interprets the HTML and displays the web page.

HTML Markup Basics: Tags, Elements, and Attributes

Tags: These are basically instructions inserted into the HTML document that give directions on how it should be formatted. Usually these instructions or elements have both an an opening or closing tag. The exception is void tags, which have no content and so don’t need a closing tag. If a tag is not void, it needs a closing tag, or whatever instruction the tag gives won't stop.

Elements: Individual components of HMTL documents. Elements are either inline or block. Block elements have a sort of “box” around them that allows you to write code with instructions on style. For example, you can set height and width.

Attributes: These are properties or values of an HTML element that modify it. For example, "href" is the attribute that tells the browswer that the text content of the element should be a live link to a website.
Handy Tips!

Computers are Stupid: You use HTML to tell the computer what to do. If you aren’t exact with your syntax, the computer will get confused about what you're telling it to do. Basically, your code won’t work.

You don’t have to memorize everything now: Right now, it’s more important to understand the underlying concepts than to memorize every tag, as you can always look up specific tags if you understand the concept behind what you’re trying to do.

Creating a Structured Document with HTML

HTML, CSS, and JavaScript Work Together on a Web Page

Like a house, a web page needs structure and style, and a way to make things inside the house work. In this metaphor, HTML is like the walls of a house- it provides structure. CSS provides style, such as the paint color of the walls. Java provides interactivity, like a garage door opener or a TV remote- it lets you change elements.

HTML Has a Tree-like Structure and Everything is a Box

Each HTML tag creates an element. These elements are like the branches of a tree. You can visualize this similarly to how you can visualize a family tree, with each offspring (element) creating a new branch. This tree-like structure is known as DOM- document object model.

When you look at a web page's HTML, you see these nested elements as a series of boxes inside one another.

When you're building your tree-like HTML structure, you can use DIV tags to define boxes. Later, you'll add style with CSS, so you want to 'label' these DIV tags for later styling. You do this by assigning class attributes- like labeling a moving box so that you know where its contents should go in the house. HTML tags can belong to multiple classes- just as you can label a box both "kitchen" and "fragile". Like labels on moving boxes, class names should be informative.

Text Editors: You Have Options

Programmers have many text editor options available for writing code. For example:

Scratchpad is a simple, easy, and beginner friendly real-time editor for HTML and CSS. You can see the changes in your code, as they would appear on a webpage, immediately on a little screen. Scratchpad also has helpful features like an automatic adding of closing tags whenever you enter an opening tag. One downside: you can't share your code.

Codepen adds Javascript, and makes it easy to share your code with a free account. It is slightly slower than Scratchpad, and a bit less user-friendly for newbies, so as a beginner, you may want to edit your simple HTML and CSS in Scratchpad and then just copy it into Codepen to save it and share it.

Sublime Text is more powerful than Scratchpad and Codepen, and has a nice feature that lets you see your works as it would look as a webpage in a full size browser page. But it's difficult to share your code from Sublime Text and is the least new-user friendly.

Adding Structure and Style With CSS

Key Features of CSS

CSS = "Cascading Style Sheets". CSS is how you take your HTML and make it look like the design you want on your website. You can use it to add structure to the page - to arrange elements on the page the way you want. And you can use it to control how things look, everything from font type to color. You do this by applying rules to the various elements in your html.

Inheritance is a key feature of CSS. It's a mechanism by which rules or properties are applied not just to any one specific element but to its descedents as well. Descendent elements "inherit" CSS property values from the ancestor elements they're enclosed within. Not everything is inheritable. Text related properties are inherited, but box related properties are not. Text related properties include things like color, font family, line height and the like. Box related properties include things like background and border elements and margins.

Inheritance allows you to avoid repetition. This is important, because it saves you from having to write the same code over and over again, so it saves you time and effort and streamlines your code. But more importantly, it can prevent errors in your code if you decide to change something later. For example, if you've used CSS to avoid repetition, you won't get stuck having to hunt for every instance of the HTML code that you need to change, and (likely) missing something that will then mess up your page.

Semantic tags: Choose the right element for the purpose - your tags should describe what they are doing. For example, use "title" to describe a title and an H tag (H1, H2, H3) to describe a heading. The tag's element name is both the semantic meaning of the content and the default style rules applied to it.

Grid and Box Structure

You can use CSS to create a grid framework for your website. The grid is the framework or base on which all of your elements will be laid out in columns and rows. You want to build asystem that gives you flexibility to make various column designs and that follows the principles of responsive design.

Flexbox is a module that you can use to efficiently lay out, align, and distribute elements on your pages, on a grid. You can use flexbox to divide your page into columns and rows and arrange them next to or on top of each other and to size them by using "containers". The "flex" references the fact that a flexbox container can alter its items' properties (like width and height) to fill the available space in the best way. This makes your pages looks its best on various types of devices and on various screen sizes. It can expand into available space or shrink.

Code, Test, Refine

The process of developing code is known as Code, Test, Refine. When you start out, you first look for natural boxes in the design. Then look for repeated styles and semantic elements. Next, start writing your html. Then apply your style sheets, starting with the biggest aspects and moving to the smallest. As you go, test your code in various browsers and formats. Then start refining your code and fixing things. Repeat.

Tip: your browser's Developer Tools provides you with ways to experiment with and test your code in the browser and is an invaluable resource. You can also use it to see how other websites are doing things, and to play with other websites' code, to test your ideas.

There are CSS and HTML testing sites that can help you figure out if your code is at least valid. You should use them.

The principal of Responsive Design is about making your site work on various browsers and platforms. You want to make sure your site looks good on everything from a desk top with a big screen using Internet Explorer, to a smart phone using Chrome. For example, for size, you can use % instead of a fixed px so that elements on your page will scale up or down depending on the size of the screen.

Telling Computers What To Do

Computers and Programming Basics

A computer is a machine that can be programmed into doing virtually any computational task you can think of, with the right program. A computer program is a precise sequence of steps given to the computer via written instructions, so that it will perform a specific computation. Computer science is the solving of problems by breaking them into pieces and writing a step-by-step list of instructions in a programming language.

We need programming languages to give instructions to computers because natural languages are too ambiguous and their grammar too imprecise. The grammar of programming languages is strict because the computer needs to be told exactly what to do with no ambiguity.

Code interpreters are programs that translate a high level language into something the computer can understand. We input our code into the interpreter and instruct it to "run" our code, and it takes the code line by line and interprets it for the computer.

Python Basics:Variables

Python is the programming lanaguge we are using in this course.

In Python, a variable is like a box you store information in, in that you use variables to give names to values. You assign value to a variable using the "=" sign, like this:

my_name = Carrie

The above is an assignment statement. Now that I've assigned the value "Carrie" to the variable "my_name", I can work with the variable. For example, if I write the following code:

print my_name

It will print out Carrie.

Variable names are case sensitive. If I later try to call the variable My_name, it will not return "Carrie" (unless I also assign "Carrie" to My_name.)

my_name = Carrie is different from 2 + 2 = 4 .

In my_name = Carrie , the equals sign means "take the value of". In 2 + 2 = 4 , the equals sign means "is the same as".

Some ways variables are useful:

  • *They give us a way to store important data.
  • *They give us a way to write readable code that the computer understands.
  • *They give us a way to change the value of something.
Python Basics: Strings

Strings are a sequence of characters surrounded by quotes. Quotes can be either single or double, as long as the string starts and ends with the same type of quote.

"I am a string." and 'I am also a string.'

but

"I am not a valid string and if you try to print me you will get a syntax error message.'

Some things to keep in mind about strings:

  1. *You can assign a string to a variable, or you can just print a string.
  2. *You can concantenate strings by using the operator +. For example, you could print the string "My name is " with the string "Carrie":
  3. "My name is " + "Carrie"
  4. and get
  5. My name is Carrie. (Note the extra space inside the string so that there's space between "My name is" and "Carrie" when it prints.)
  6. *When working with numbers, remember that 500 is a number, while "500" is a string. So 500 + 2 will print out as 502 (Python will add the two numbers), while "500"+"2" will print out as 5002 (Python will concantenate the two strings).
